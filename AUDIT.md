## 🔴 Критичное — контраст текста (WCAG 2.1 AA)

Норма AA: обычный текст ≥ 4.5:1, крупный (≥18px или ≥14px bold) ≥ 3.0:1.

| Цвет | Контраст с фоном `--bg` | Вердикт |
|---|---|---|
| `--ink` #1F1A14 | 16.3 | ✅ |
| `--ink-2` #5A4F3F | 7.5 | ✅ |
| `--c-blue` #2E5C8A | 6.6 | ✅ |
| **`--ink-3` #8A7E69** | **3.76** | ⚠️ ниже 4.5 — годится только для крупного текста |
| **`--gold` #B68B2F** | **2.94** | ❌ ниже 3.0 — не проходит даже для крупного |

**Проблема 1 — золотой текст `#B68B2F` (2.94).**
Используется в особых днях лунного календаря («День Дхармапал»), золотых лейблах. Текст плохо читается на кремовом фоне.
*Рекомендация:* для золотого ТЕКСТА затемнить до ≈`#856518` (даст ~4.6). Золото в фоне-плашках и полосках оставить как есть.

**Проблема 2 — `--ink-3` #8A7E69 (3.76).**
Используется для описаний, дат, процентов размером 11–13px (мелкий текст). Для такого размера норма — 4.5.
*Рекомендация:* затемнить `--ink-3` до ≈`#6E6350` (даст ~4.6). Это глобально поднимет читаемость подписей.

## 🟡 Среднее — чистота кода

**Проблема 3 — хардкод `#000`.**
В `.btn-small:hover` и `.btn-primary:hover` фон уходит в чистый чёрный. Это рудимент старого чёрного дизайна кнопок (до сине-градиентного CTA). Выбивается из палитры.
*Рекомендация:* заменить на затемнённый синий, напр. `#1F3F5F` (уже используется в градиенте CTA).

**Проблема 4 — синие оттенки мимо переменных.**
`#1F3F5F`, `#3A6FA3` (градиент CTA), `rgba(46,92,138,.92)` (фон bell-banner) — захардкожены.
*Рекомендация:* вынести в переменные `--c-blue-dark`, `--c-blue-light`.

**Проблема 5 — `#fff` 5×.**
Белый текст на цветных подложках повторяется литералом.
*Рекомендация:* ввести `--on-accent: #fff`.

## 🟢 Косметика / на подумать

**Проблема 6 — 49 КБ base64-логотипа inline.**
Логотип-мандала встроен в HTML в 3 разрешениях (favicon 64, apple-touch 180, header 192). Раздувает каждую загрузку страницы.
*Компромисс:* можно вынести в отдельные `.png` — браузер закеширует независимо от HTML. Но для single-file-подхода (приложение = один файл, легко переслать/открыть) текущее решение осознанно. Менять только если важна скорость первой загрузки.

**Проблема 7 — `innerHTML` 10×.**
Для собственных данных приложения безопасно. Пользовательский ввод (заметки) — экранируется. Поля `selfCheck`/`dedication` из wizard'а стоит проверить, что они тоже идут через `escapeHtml` при рендере (если рендерятся в HTML).

## Приоритет исправлений

1. **Сначала** — контраст (проблемы 1, 2): затронет читаемость для всех, особенно на ярком экране телефона. 2 строки в `:root`.
2. **Потом** — хардкод `#000` (проблема 3): косметика, но бросается в глаза дизайнеру.
3. **По желанию** — переменные для синих оттенков и белого (4, 5): чистота, не влияет на пользователя.
4. **Опционально** — вынос логотипа из base64 (6): только если важна скорость загрузки.

## Что НЕ является проблемой

- Single-file без сборки — оправдано для масштаба проекта.
- Зависимость от 2 CDN (шрифты, тибетский календарь) — есть graceful fallback: при недоступности CDN приложение не падает.
- Размер 183 КБ — нормально для офлайн-способного приложения с полными текстами молитв и логотипом.
