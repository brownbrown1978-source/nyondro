# Контекст проекта · Нёндро

Этот файл — общий брифинг для второго пользователя Claude и подсказка для самого Claude в начале новой сессии. Если ты Claude — прочитай его перед началом работы.

## Цель проекта

Сделать и поддерживать персональный **трекер раздела «Драгоценная человеческая жизнь» (ДЧЖ)** традиции Дзогчен Лонгчен Ньинтиг. Не «универсальное приложение для всех» — конкретный инструмент для двух практикующих, которые работают над ним вдвоём через Claude.

## Уже принятые решения

- **Платформа** — один HTML-файл `ngondro.html`, без сборки и зависимостей. Хранение в `localStorage` браузера.
- **Стиль интерфейса** — журнал тунов в духе Strava: лента карточек + дашборд недели сверху + кнопка «новый тун».
- **Цветовая палитра и иконка** — пять цветов флага дзогчен (синий / белый / жёлтый / красный / зелёный) как пять мудростей.
- **Охват данных v1** — только раздел 1 (ДЧЖ): 34 признака + 3 причины. Остальные разделы пока не включены.
- **Стратегии счёта** — обе поддержаны: классическая (1000 кругов подряд, 28 кр./признак, 16 кр./причина) и 25/75 (7+21 / 4+12 за два прохода). Цель раздела = ровно 1000 кругов.
- **Техника начитывания** — **только Техника 1** (три захода с подразбивкой). Селектор техник убран из формы; вместо него встроена раскрывающаяся подсказка по структуре трёх заходов с двумя колонками по стратегиям.
- **Дефолт «Кругов» в форме** — 10 для классической, 7 для 25/75 (соответствует Технике 1).

## Где живёт «истина»

- **Цифры подсчёта** (28 / 16 / 1000) — `pdfs/02-recitation-techniques.pdf` и `pdfs/01-calculations-by-attributes.pdf`.
- **Список 34 признаков и 3 причин** — `nyondro_razdel1_guide.md`.
- **Толкования и фрагменты для размышления** — текст Паттрула в `patrul-rinpoche-words-of-my-teacher.md` (большой файл) и сжатые выдержки прямо в `ngondro.html` (объект `READING`).
- **Структура туна, подготовка ума** — `pdfs/03-tun-structure.pdf`.

## Что считается готовым

См. `TASKS.md`, секция «Готово».

## Принципы работы

- **Не ломать существующее** — приложение уже используется. Любая правка `ngondro.html` должна сохранять обратную совместимость со схемой `state.tuns` (поля: `id, date, length, groupKey, idx, tech, circles, prostr, feel, note`).
- **Решения фиксируем здесь** — если в обсуждении принимается архитектурное решение (например «убираем все техники кроме первой»), оно дописывается в раздел «Уже принятые решения» этого файла.
- **Задачи живут в `TASKS.md`** — сессионный TodoList в чате не виден второму пользователю. Любая новая задача дублируется в `TASKS.md`.
