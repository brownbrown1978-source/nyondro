# Контекст проекта · Нёндро

Этот файл — общий брифинг для второго пользователя Claude и подсказка для самого Claude в начале новой сессии. Если ты Claude — прочитай его перед началом работы.

## Цель проекта

Сделать и поддерживать персональный **трекер раздела «Драгоценная человеческая жизнь» (ДЧЖ)** традиции Дзогчен Лонгчен Ньинтиг. Не «универсальное приложение для всех» — конкретный инструмент для двух практикующих, которые работают над ним вдвоём через Claude.

## Уже принятые решения

- **Платформа** — один HTML-файл `ngondro.html`, без сборки и зависимостей. Хранение в `localStorage` браузера. Данные практики — индивидуальные, никакого облачного синхрона (никогда).
- **Стиль интерфейса** — журнал тхунов в духе Strava: лента карточек + дашборд недели сверху. Две CTA-кнопки сверху: большая **«Начать тхун»** (открывает wizard) и маленькая **«Записать задним числом»** (старый quick-log модал).
- **Цветовая палитра и иконка** — пять цветов флага дзогчен (синий / белый / жёлтый / красный / зелёный) как пять мудростей.
- **Охват данных** — все 6 разделов внешних предварительных по PDF «Практика нёндро в туне» (Дзогчен Шри Сингха). Каждый = 1000 кругов (±15).
- **Стратегии счёта** — обе поддержаны: классическая (1000 кругов подряд) и 25/75 (за два прохода). Цель каждого раздела = 1000 ± 15.
- **Техника начитывания** — только Техника 1.
- **Дефолт «Кругов» в форме** — 10 для классической, 7 для 25/75.

## UX-архитектура (после 2026-05-13)

- **Вкладок ровно 4**: Журнал · Прогресс · Практика нёндро · Настройки. Не добавлять «Подготовка» как пятую вкладку — подготовка живёт внутри wizard'а.
- **Wizard «Начать тхун»** — единый линейный поток из 5 шагов (Приготовления → Подготовка → Обет → Медитация → Завершение). Реализован как fullscreen modal overlay. Не разбивать на разные вкладки.
- **Quick log** (`#btn-add`) — остаётся для записи задним числом, не удалять.
- **Вкладка «Практика нёндро»** — два sub-tab'а: «Общие практики» (8 блоков LITURGY) и «Раздельные практики» (per-признак reader с полным текстом начитывания из PDF + размышления Паттрула).

**Раздел 4 — особый случай**: визуально текст разделён на стансы, но счётчик един для всего раздела (`ATTRS.karma.isMonolith === true`). Это решение Ильи (2026-05-13).

**Порядок ядов в s3** строго по PDF: гнев → зависть → жадность → гордость → неведение → привязанность. Не менять.

## Бэлл (Web Audio)

- Синтез поющей чаши через 5 синусоидальных гармоник (1.0 / 2.0 / 2.76 / 4.10 / 5.40 × fundamental).
- Экспоненциальное затухание + 4.2 Hz LFO для «дыхания».
- Звенит на конце шаматхи (короткий, 4 с, pitch 523 Hz) и на конце медитации (длиннее, 7 с, pitch 392 Hz).
- Если синтез звучит дёшево — заменить на встроенный base64-сэмпл (TASKS.md).

## Принципы работы

- **Не ломать существующее** — приложение уже используется. Любая правка `ngondro.html` должна сохранять обратную совместимость со схемой `state.tuns` (поля: `id, date, length, groupKey, idx, tech, circles, prostr, feel, note`).
- **Решения фиксируем здесь** — если в обсуждении принимается архитектурное решение (например «убираем все техники кроме первой»), оно дописывается в раздел «Уже принятые решения» этого файла.
- **Задачи живут в `TASKS.md`** — сессионный TodoList в чате не виден второму пользователю. Любая новая задача дублируется в `TASKS.md`.
