# Нёндро — протокол правки читалки (обязателен)

Принят по итогам совета. Цель: править веб-читалку сборника «Путь Будды» (`ngondro.html`) без поломок вёрстки и без путаницы версий.

## Архитектура — НЕ менять
- `ngondro.html` остаётся **single-file** (текст + вёрстка вместе). Это фича: атомарность священного текста, офлайн на ретрите, простой деплой.
- **НЕ выносить** текст в `content.md`/`content.json`. Вёрстка здесь пиксельно вшита в разметку (построчный жирный/курсив, убывающие «А», тибетские слоги, центровка) — текст и форма неразделимы. Конвертация = отрицательная окупаемость + сломает точечные правки и офлайн.
- Картинки уже вынесены в `sb/*.png` (ссылки `<img src>`). Это правильный и достаточный вынос. **base64 не возвращать.**

## Как править
- **Только якорная literal-замена** конкретного фрагмента: уникальная строка «было → стало». **НИКАКИХ пакетных/многострочных regex** — именно это склеило заголовки.
- Перед правкой — прочитать актуальный файл (источник правды = файл в репо, не сообщения чата).
- Менять ровно запрошенное; не трогать соседний текст/вёрстку/картинки; не «причёсывать» по своей инициативе; не менять порядок глав.
- Одна правка за раз, маленькими порциями. Показывать дифф «было → стало» по изменённому месту.

## Обязательная проверка (приёмка)
- **Эталон — `reference/Путь Будды (эталон).pdf`.** Сверять результат рендером (playwright) против страницы PDF до/после. Без этого якорная замена молча разойдётся с каноном.
- Целостность: баланс `<b>`/`<i>`, баланс `{`/`}` в CSS, число `assert(` не упало.
- `git`: один смысловой фикс = один коммит (мгновенный откат).

## Деплой и версии
- Пуш в `brownbrown1978-source/nyondro` (ветка `main`), GitHub Pages. Токен в `.github-token` (НЕ коммитить).
- Service worker (`sw.js`) бампит версию кэша — обновление подхватывается само. При показе «старого» — жёсткий релоад / пересоздать PWA-ярлык.
- Карта людей: Илья (источник) → Аркадий (правит, пуш-таргет выше) → Павел (downstream через live-ссылку). Следить, что downstream видит ту же ветку.

## Рендер в песочнице (для сверки)
- chromium playwright: не хватает только `libXdamage.so.1` → `apt-get download libxdamage1`, распаковать, `LD_LIBRARY_PATH=/tmp/libs ... --no-sandbox`.
- Секция `.main-pane` по умолчанию `display:none` — форсировать `display:block!important`; снимать по якорю с `clip`.
