# Нёндро · ДЧЖ

Трекер раздела «Драгоценная человеческая жизнь» традиции Дзогчен Лонгчен Ньинтиг.
Журнал тхунов, прогресс по 34 признакам и 3 причинам, фрагменты из «Слов моего всеблагого учителя» Патрула Ринпоче.

**Открыть приложение:** [ngondro.html](./ngondro.html)
(после публикации через GitHub Pages — ссылка станет вида `https://<ник>.github.io/<репо>/ngondro.html`)

## Что внутри

- **Wizard «Начать тхун»** — единый поток из 5 шагов: Приготовления → Подготовка (с регулируемой шаматхой 3–30 мин) → Обет → Медитация (с таймером и счётчиком кругов) → Завершение (самопроверка + посвящение).
- **Web Audio бэлл** — мягкий тон поющей чаши (синтез, без внешних сэмплов). Звенит на конце шаматхи и в конце тхуна.
- **Журнал тхунов в стиле Strava** — карточки с датой, длительностью, признаком, кругами, чувством, заметкой.
- **Прогресс** по всем 6 разделам внешних предварительных, каждый = 1000 кругов.
- **Вкладка «Практика нёндро»** — полный текст всех молитв и строк начитывания из PDF «Практика нёндро в туне» (Дзогчен Шри Сингха):
  - 8 общих практик: молитва мастерам, прибежище, 4 безмерных, Бодхичитта, семистрочная, сущностные строки, обет тхуна, посвящение заслуг
  - Полные 15-строчные блоки начитывания для каждого из 34 признаков Раздела 1
  - Полные блоки для разделов 2–6 (8 оснований + 18 примеров непостоянства; 6 ядов; монолит кармы; 7 свобод нирваны; 3 умения следования учителю)
- **Размышления Паттрула Ринпоче** по каждому признаку — для подготовки ума.
- **Запись задним числом** — старый quick-log сохранён как secondary CTA для случаев, когда тхун не записан в реальном времени.
- **Две стратегии**: классическая (1000 кругов подряд) и 25/75.
- **Тесты** на структуру данных и миграцию (50+ assertions, прогоняются при каждой загрузке).
- Экспорт/импорт JSON, всё хранится локально (`localStorage`). Никакого облачного синхрона — данные практики индивидуальные.

## Стек

Один `.html` файл. Без сборки, без зависимостей. Просто открыть в браузере.

## Источники данных

- **PDF «Практика нёндро в туне»** (Дзогчен Шри Сингха) — главный канон для всех 6 разделов и литургии.
- Гайд по разделу ДЧЖ — Центр «Дзогчен Шри Сингха».
- «Слова моего всеблагого учителя», Патрул Ринпоче (перевод Фариды Маликовой).
- Расшифровка консультации с Пашей по правилам выполнения.

## Совместная работа двух Claude-пользователей

Cowork-проекты Claude расшарить нельзя; шаринг обычных Claude-проектов есть только в Team/Enterprise. На тарифе **Max** прямой шаринг недоступен. Поэтому мы используем **этот GitHub-репозиторий как общий источник знаний** для двух независимых Claude-проектов.

### Один раз — настройка владельцем репо

1. Зарегистрироваться/войти на [github.com](https://github.com), создать **публичный** (или приватный, если друг будет collaborator) репо, например `nyondro-dchz`.
2. Загрузить туда всё содержимое этой папки: `ngondro.html`, `README.md`, `CONTEXT.md`, `TASKS.md`, `knowledge/`.
3. (Если приватный) Settings → Collaborators → добавить друга по логину/email.
4. Опционально: Settings → Pages → Branch `main` / `/ (root)` → Save → получаете публичный URL для самого приложения.

### Каждому из двоих — подключение в свой Claude

1. Открыть [claude.ai](https://claude.ai) → **Projects** → **Create Project** → название «Нёндро».
2. Внутри проекта: **Add knowledge** → выбрать **GitHub** → авторизовать → выбрать тот самый репозиторий → подтвердить.
3. После синхронизации Claude увидит все файлы (`ngondro.html`, документы из `knowledge/`, `CONTEXT.md`, `TASKS.md`) и сможет с ними работать.
4. По желанию: открыть проект в **Cowork** (десктопное приложение Claude → выбрать этот проект → начать сессию). Cowork даст файловый доступ — приложение можно будет открывать локально и тестировать.

### Как это работает на практике

- Каждый ведёт **свои чаты** с Claude — они не видны другому. Но у обоих Claude видит **одно и то же** содержимое репо.
- Когда один из вас (или Claude в его сессии) меняет файл и пушит в GitHub — у второго Claude обновится контекст после следующей синхронизации проекта (обычно автоматически).
- **`CONTEXT.md`** — это брифинг для второго пользователя и для самого Claude. В нём фиксируем уже принятые решения. Любая существенная договорённость — туда.
- **`TASKS.md`** — общий список задач. Сессионный TodoList в чате не виден другому, поэтому всё важное переносим в этот файл.

### Что отличается от настоящего шаринга

- Невозможно увидеть **живой чат** друга и его сессионные задачи — только snapshot через ссылку «Share chat» (read-only).
- Не синхронизируется **history тхунов в самом приложении** — данные практики у каждого свои в `localStorage` его браузера.
- Если эти ограничения станут мешать — следующий шаг: либо переход на Team-план для одного из вас (тогда настоящий шаринг проектов), либо лёгкий бэкенд для совместного журнала тхунов (Firebase/Supabase, ~1–2 ч работы).
